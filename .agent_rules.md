# Agent Rules & Guidelines - Personal Wealth App

This file governs the research, mathematical modeling, and documentation standard for this repository. Any agent modifying or adding files to this project must strictly adhere to these guidelines.

---

## 🎯 1. Core Objectives & Mission

* **Focus on Wealth**: Every feature, calculator, tracker, and simulation must be centered around wealth creation, optimization, preservation, and education.
* **Indian Market Specifics**: Projections, formulas, calculations, and simulations should default to the Indian economic ecosystem (INR ₹ currency, Indian tax brackets, Nifty returns, domestic inflation patterns).
* **Developer Ready**: All research must be structured logically, making it easy to translate into a React Native codebase (TypeScript, component-based logic, clean formulas).

---

## 📊 2. Indian FinTech Assumptions & Benchmarks

Unless overridden by user preference in `QUESTIONS_ANSWERS.md`, use these verified standard benchmarks:

* **Long-Term Inflation**: Default to **5.5%** (Historical CPI range is 5%-6%).
* **Equity Mutual Funds / Nifty 50 Index Return**: Default to **12% CAGR** (Conservative long-term equity growth standard).
* **Mid-Cap / Small-Cap / Active Equity Return**: Default to **15% CAGR** (Aggressive growth standard).
* **Debt / Fixed Deposits / EPF**: Default to **6.5% - 7.5% CAGR**.
* **Gold**: Default to **8% CAGR**.
* **Real Estate Growth (India)**: Default to **8% CAGR** (excluding rental yield, which typically stands at 2-3% in major Indian tier-1 cities).
* **Taxation (Standard Defaults)**:
  * **Short-Term Capital Gains (STCG) on Equity**: 15% (or latest standard).
  * **Long-Term Capital Gains (LTCG) on Equity**: 10% for gains exceeding ₹1 Lakh per financial year (without indexation).
  * **Slabs**: Default to the New Tax Regime (simplified slabs without deductions).

---

## 📐 3. Mathematical Formatting Rules

* **Explicit Formulas**: Always document the raw mathematical formulas in LaTeX format ($$Formula$$) before writing pseudo-code.
* **Variable Explanations**: Every variable in a formula must be explicitly defined with units and bounds.
* **Code Implementation Proving**: Include a brief JavaScript/TypeScript function implementing the math directly in the markdown file so developers can copy-paste it.

Example layout:
```markdown
### Compound Interest Formula
$$A = P \left(1 + \frac{r}{n}\right)^{nt}$$

*Where:*
* **A** = Future value of the investment
* **P** = Principal investment amount
* **r** = Annual interest rate (decimal)
* **n** = Number of times interest is compounded per year
* **t** = Time the money is invested (years)
```

---

## 📝 4. Research Document Structure

Every file in `research/` must follow this structure:

1. **Executive Summary**: 2-3 sentence overview.
2. **Competitor Benchmarking**: What do existing apps do well? What are they missing? (backed by Google searches).
3. **Core Features & User Flow**: How the feature behaves in our app.
4. **Mathematical Engine**: Formulas, default parameters, and TypeScript pseudo-code.
5. **UI/UX Premium Design Concepts**: Layout descriptions, visual design tokens (colors, animations).
6. **React Native Implementation Notes**: Packages to use (e.g., standard libraries), state structure, and key considerations.
7. **Monetization Triggers**: How this specific feature can drive premium conversions.

---

## 🚀 5. Modular Token Preservation

* **No Redundancy**: Do not duplicate core concepts (e.g., compounding formula explanation) across multiple files. Link to the originating document instead.
* **Conciseness**: Keep documentation dense, highly formatted with tables, checklists, and visual descriptions, avoiding long prose.

---

## 🔒 6. Privacy & Offline-First Architecture

* **Zero Backend Costs / Offline Storage**: Do not research or propose server sync, external databases, cloud users, or backend logic. Everything is kept local-only using local databases (e.g., React Native MMKV).
* **Absolute Privacy Stance**: Our marketing and design must emphasize that **"not even our team or DB admin can see your data."** No telemetries that compromise user balance privacy.

---

## ⚖️ 7. SEBI Compliance & Safety

* **No Stock Tips / Advisory**: Do not write algorithms that recommend specific stocks, mutual funds, or brokers. Projections must use generalized assets (e.g., "Equity Mutual Funds at 12% CAGR") rather than naming specific mutual fund houses.
* **Pure Education**: Frame all content, simulators, and daily stories as educational gamification, completely immune to financial advisory regulatory (SEBI) requirements.
